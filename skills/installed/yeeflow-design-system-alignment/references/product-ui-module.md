02. PRODUCT UI MODULE

Purpose
Use this module for:
- website UI
- product UI
- dashboards
- forms
- portals
- settings pages
- application shells
- UI mockups
- product screenshots
- design system surfaces

Apply the Master Yeeflow Design Standard first, then apply this module.

Primary design objective
Create a production-quality user interface that is:
- modular
- reusable
- scalable
- easy to maintain
- visually coherent
- grounded in realistic product behavior
- suitable for enterprise or production environments

Product UI should feel:
- calm
- structured
- trustworthy
- clear
- task-oriented
- operational
- modern
- polished
- scalable

Product UI should not feel:
- like a campaign poster
- decorative
- glow-heavy
- neon-heavy
- concept-only
- visually noisy
- disconnected from real workflows

Layout principles
Design the interface as part of a coherent product ecosystem.
Use strong shell and content structure, such as:
- top header
- navigation rail or sidebar when relevant
- main content region
- grouped sections
- reusable layout blocks
- responsive containers

The first visible area should be immediately useful.

Information hierarchy
The UI must clearly communicate:
1. what the page is for
2. what should be noticed first
3. what the user can do immediately
4. what is active, pending, recommended, or important
5. what can be explored next

Use hierarchy through:
- typography
- spacing
- grouping
- alignment
- contrast
- scale
- section sequencing

Design system rules
Rely on:
- neutral surfaces
- subtle borders
- restrained depth
- clear text hierarchy
- reusable patterns
- predictable component behavior

Do not rely on:
- loud gradients
- heavy shadows
- decorative flourishes
- random card styles
- overly expressive effects

Color usage rules for product UI
- Blue is the primary action and anchor color
- Purple may be used for controlled AI emphasis
- Pink should be rare and tertiary
- Most surfaces should remain neutral
- Text should prioritize readability and clarity
- Strong gradients should be limited to small highlight moments only

Component guidance
Prefer reusable primitives such as:
- cards
- section headers
- buttons
- tabs
- search inputs
- filters
- lists
- panels
- stat blocks
- tables
- chips and badges
- wrappers and layout containers

Interaction guidance
Interactions should be:
- subtle
- clear
- useful
- predictable

Hover, focus, and active states should be visible but restrained.

Content and copy rules
- Use realistic labels
- Use specific and concise copy
- Use practical business scenarios
- Keep interactions understandable
- Avoid filler metrics and fake content

Responsive behavior
The output should work across:
- desktop
- tablet
- mobile

Responsive behavior should feel designed, not merely shrunk.

Logo and typography rules
- Use official logo assets only
- Do not recreate the Yeeflow wordmark for product UI
- Use clean, scalable sans-serif typography
- Do not use Playfair Display in interface typography

Final rule
The result must feel like a real product UI that a frontend team could implement and evolve.
