Service Workload Estimation Excel Standard

Purpose
Use this standard for Yeeflow service-team workload estimation deliverables created as Excel files (.xlsx).

Output format
- Excel workbook (.xlsx)
- one sheet only
- sheet name must be: Workload Estimation

Primary objective
Create workload estimation files that are:
- clear
- structured
- printable
- easy to review
- easy to export to PDF
- realistic
- reusable across projects

The estimation should feel:
- professional
- calm
- structured
- enterprise-ready
- operationally credible

Required main sections
Every workload estimation must use these 4 top-level sections:

1. Section 1: Initiation and Management
2. Section 2: System Implementation
3. Section 3: Testing and Verification
4. Section 4: Documentation and Go Live

Section 2 sub-section rule
- Section 2 may stay as one direct section with task rows underneath
- If the business requirement is more complex, Section 2 may be broken into sub-sections
- Only add sub-sections when they improve clarity
- Do not over-fragment simple estimates

Required columns
Each task row should contain only these columns:
- Task / Item
- Description
- Owner / Role
- Effort (man-days)

Do not add unnecessary extra columns to the main estimation sheet unless explicitly requested.

Required calculation rules
- each main section must have a subtotal row for man-days
- the workbook must include a final total man-days row at the end
- effort values should be numeric and easy to sum
- subtotal and total rows must be clearly visible

Sheet layout rules
- keep all content in one sheet only
- organize rows using section headers, optional sub-section headers, task rows, section subtotal rows, and one final total row
- keep the layout easy to scan
- use clean row grouping logic
- use consistent spacing and styling

Print and PDF export rules
The workload estimation sheet must be optimized for PDF export and printing.

Requirements:
- all columns should fit within one page width
- no column should spill into a second printed page
- keep the column set narrow and controlled
- use landscape orientation where appropriate
- use wrapping carefully so the sheet remains readable
- size columns intentionally to avoid column splitting across pages

Design rules
Apply the Yeeflow visual system:
- modern
- calm
- structured
- enterprise-ready
- restrained use of brand color
- clear hierarchy
- easy scanning

Use Yeeflow palette as the dominant palette:
- #146FF6
- #7C4DFF
- #E85ECF
- #333333
- white / soft neutrals

Recommended visual structure
- title row at the top
- small project/context details area
- main table header row
- section header rows clearly styled
- optional sub-section rows styled lighter than section rows
- subtotal row after each section
- one final total row at the end

Review requirements
Before finalizing, check:
- is there only one sheet called Workload Estimation?
- are the 4 required sections present?
- are sub-sections used only when needed?
- are only the required task columns present?
- does each section have a subtotal?
- is there a final total man-days row?
- will the sheet export cleanly to PDF with all columns inside one page width?
