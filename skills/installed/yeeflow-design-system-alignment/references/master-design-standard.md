01. MASTER YEEFLOW DESIGN STANDARD

Purpose
Use this standard for every Yeeflow design or content-generation task in 2026.
Apply this file first before any material-specific module.

Brand and product positioning
Yeeflow should be presented as a governed enterprise AI application and execution platform.

All outputs should align with these strategy themes:
- AI-powered build
- AI-powered use
- AI-powered execution
- platform extensibility
- AI governance and control
- AI work orchestration and reuse

Core design intent
The output should feel:
- modern
- calm
- structured
- trustworthy
- task-oriented
- polished
- scalable
- intentionally designed
- enterprise-ready
- product-led

The output should not feel:
- generic
- cluttered
- overdecorated
- consumer-chat-like unless explicitly required
- overly futuristic
- hype-heavy
- experimental without practical UX
- disconnected from real business use

Visual quality should come from:
- spacing
- hierarchy
- proportion
- alignment
- restraint
- consistency
- thoughtful component patterns

Do not rely on:
- heavy shadows
- loud gradients
- decorative noise
- random visual effects
- abstract AI imagery with no product meaning

Brand color system
Primary: #146FF6
Secondary: #7C4DFF
Tertiary: #E85ECF
Neutral dark: #333333

Supporting neutrals
- white and soft near-white backgrounds
- light gray surfaces
- subtle border grays
- readable secondary text grays

Color rules
- Use blue as the main anchor color
- Use purple as controlled AI emphasis
- Use pink sparingly as tertiary highlight
- Use neutral surfaces to preserve readability and structure
- Use dark neutral for core text and strong brand seriousness
- Do not apply all bright colors equally in one design

Logo rules
- Use approved Yeeflow logos only
- On light backgrounds, use blue logo or blue symbol with dark wordmark
- On dark backgrounds, use white logo
- Do not stretch, rotate, recolor, outline, or decorate the logo
- Do not place logo on low-contrast backgrounds
- Editorial wordmark styling is only for approved campaign/editorial use, not default product UI

Typography rules
- Product and website UI should use clean, scalable sans-serif typography
- Editorial wordmark treatment may use Playfair Display only when intentionally requested for campaign or editorial materials
- Do not use editorial typography for app navigation, labels, controls, forms, or dashboard UI

Content rules
- Use realistic product behavior
- Use realistic labels and business examples
- Use practical operational scenarios
- Avoid lorem ipsum
- Avoid vague slogans
- Avoid fake futuristic claims
- Avoid overpromising AI autonomy
- Keep the content connected to actual Yeeflow platform value

Roadmap realism rules
Keep the output aligned with Yeeflow’s realistic 2026 maturity path:
- H1: expand AI capability and platform openness
- Q3: governance, observability, lifecycle, testing, context foundation
- Q4: reuse, orchestration, memory, human review, and simple multi-component creation

Do not imply advanced capabilities earlier than the roadmap supports.

System consistency rules
Every output should feel like part of one Yeeflow system.
Do not produce isolated one-off visuals with inconsistent tone, color, structure, or product logic.

Default quality bar
Aim for work that is:
- above generic template quality
- below unnecessary visual experimentation
- suitable for real product, marketing, sales, or event use
- coherent with Yeeflow’s 2026 strategy and messaging
