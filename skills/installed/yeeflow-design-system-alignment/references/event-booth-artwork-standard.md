Yeeflow Event Booth Artwork Standard

Purpose
Use this standard for Yeeflow exhibition booth artwork, including large wall graphics and table-front panels for events such as Asia Tech x Singapore.

This standard defines how Yeeflow should look in large-format event environments so the booth feels:
- modern
- calm
- structured
- enterprise-ready
- readable from distance
- credible for enterprise technology audiences

Core principle
The booth should present Yeeflow as a governed enterprise AI application and execution platform.
It should not look like a generic AI startup booth or a decorative tech-art installation.

1. Booth storytelling model

Treat the booth as one connected narrative across multiple surfaces.

Recommended booth roles
- Hero wall: what Yeeflow is
- Operating model / explanatory wall: how Yeeflow creates value
- Connected enterprise / proof wall: why Yeeflow fits real business environments
- Table front: simple close-range anchor statement

Each surface should have one main job.
Do not make every surface try to explain everything.

2. Design tone

All booth artwork should feel:
- premium
- calm
- structured
- enterprise-ready
- product-led
- credible
- readable at a glance

Avoid:
- clutter
- dense paragraphs
- small unreadable text
- generic futuristic AI art
- neon-heavy effects
- overdecorated backgrounds
- too many competing diagrams

3. Large-format readability rules

Because walls are viewed from distance:
- use large headlines
- keep supporting text short
- prefer strong hierarchy over detail density
- reduce the number of visual elements
- do not rely on small labels or tiny UI details
- make the main message understandable within 3 to 5 seconds

Recommended content density
- Hero wall: lowest density
- Explanatory wall: medium density
- Proof wall: medium density
- Table front: very low density

4. Wall type A - Hero wall

Purpose
The hero wall should establish Yeeflow’s category, value, and first impression immediately.

Recommended content structure
- Yeeflow logo
- one major headline
- one short support line
- one dominant visual anchor
- optional short proof-point row

Design rules
- most brand-forward wall
- least crowded wall
- one strong focal composition
- minimal detail
- big readable typography
- keep the message visible from distance

Recommended visual behavior
- large clean composition
- simple connected-platform or orchestration visual
- restrained background
- strong whitespace

5. Wall type B - Explanatory / operating model wall

Purpose
Explain how Yeeflow creates value in a more structured way.

Recommended content structure
- one clear title
- one short support line
- one structured flow or framework visual
- 3 to 4 concise supporting proof points

Design rules
- more explanatory than the hero wall
- still visually calm and readable
- use a structured flow, not a crowded infographic
- avoid too many equal-priority elements

Recommended visual behavior
- build -> use -> execute style flow
- controlled supporting labels
- simplified platform narrative
- clean diagram blocks

6. Wall type C - Connected enterprise / proof wall

Purpose
Show why Yeeflow fits real enterprise environments.

Recommended content structure
- one title
- one short support line
- one central node / system graphic
- surrounding proof-point modules

Typical proof-point themes
- connected applications
- workflows
- APIs and services
- data
- external systems
- governance
- reuse
- portability
- enterprise integration

Design rules
- concrete and credible
- visually structured
- show platform connection, not decorative complexity
- use present-tense product proof where possible

7. Table front standard

Purpose
Provide close-range brand reinforcement.

Recommended content
- Yeeflow logo
- one short line only

Do not use:
- dense text
- multi-step diagrams
- long explanations
- many proof points

Recommended design behavior
- simple
- bold
- clean
- brand-consistent
- readable from close and medium distance

The table front should feel like an anchor, not a mini wall graphic.

8. Background style rules

Backgrounds should be:
- clean
- controlled
- premium
- supportive, not dominant

Use:
- soft gradient fields when needed
- restrained color transitions
- clean negative space
- subtle structure
- minimal visual noise

Avoid:
- heavy tech trails
- excessive glow effects
- overbusy patterns
- dramatic sci-fi backgrounds

For hero walls, a more premium gradient field may be used if it remains calm and clean.
For explanatory and proof walls, keep the background even more restrained.

9. Color rules

Use the dominant Yeeflow palette:
- #146FF6
- #7C4DFF
- #E85ECF
- #333333
- white / soft neutrals

Rules
- keep the palette controlled
- blue should remain the anchor color
- purple and pink should be accents, not visual overload
- avoid teal/green drift
- do not over-darken the full booth unless the concept strongly requires it

10. Logo rules

Use the official Yeeflow logo assets only.

Rules
- light background: use the standard full logo
- dark background: use the white full logo
- symbol-only versions only when clearly justified by space or design intent
- do not redraw or restyle the logo
- keep strong contrast against the background
- do not let the logo compete with the main wall headline

11. Layout rules

Use strong layout zones:
- header / brand zone
- message zone
- visual explanation zone
- optional proof-point zone

Each wall should have a clear hierarchy:
- first read
- second read
- supporting detail

Do not give all elements equal visual weight.

12. Print production rules

For large-format event artwork:
- build final artwork at the correct trim size
- add required bleed on all sides
- keep text and logos inside a safe area
- avoid placing critical content too close to edges
- ensure exported artwork is suitable for print production

Recommended safety rule
- keep major text and logos comfortably inside the trim edge
- allow enough margin for mounting tolerance and visual breathing room

13. Event-message alignment

Event artwork should align with Yeeflow’s 2026 strategy:
- enterprise AI application and execution platform
- AI-powered build
- AI-powered use
- AI-powered execution
- platform extensibility
- AI governance and control
- AI work orchestration and reuse

Avoid event messaging that sounds like:
- generic AI transformation
- vague autonomy claims
- consumer AI branding
- overly abstract innovation language

Preferred messaging should emphasize:
- real business systems
- governed AI
- practical operational work
- connected execution
- extensibility
- enterprise fit
- reusable platform value

14. Final decision rule

When designing booth artwork:
- Hero wall = strongest positioning statement
- Explanatory wall = clearest operating model
- Proof wall = strongest enterprise credibility
- Table front = simplest anchor message

All surfaces should feel like one Yeeflow system, not unrelated posters.
