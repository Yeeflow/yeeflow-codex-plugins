# Yeeflow Copilot Import Export Operator

Use this skill for Yeeflow Copilot `.yaic` package work.

## Export Workflow

1. Confirm the source Yeeflow Copilot category and Copilot names.
2. Query or inspect live Copilots and collect IDs.
3. Export each Copilot using the authenticated Yeeflow session or known API route.
4. Save files under the requested folder. Prefer `output/Copilot`.
5. Include companion icons in `output/Copilot/icons` when the user asks to include icon images.
6. Validate:
   - expected file count
   - non-empty files
   - readable outer JSON wrapper
   - `Type = 1`
   - `Name`, `Description`, `IconUrl`, and `PackageJson` present
   - no accidental duplicate or stale export

## Known Yeeflow Export Notes

- `.yaic` files are JSON wrappers with an opaque `PackageJson` payload.
- Copilot wrappers use `Type = 1`; AI Agent wrappers use `Type = 0`.
- Copilot normalized draft content uses `Instructions`; AI Agent draft content uses `Prompt`.
- Final `.yaic` packages should be generated by Yeeflow import/export APIs, not by manually constructing `PackageJson`.
- Reusable Copilot packages should usually use `Components: []`; store recommended tool behavior in the Instruction unless real bindings exist.

## Safety Rules

- Do not import into Yeeflow until the user explicitly asks.
- Do not overwrite, delete, rename, or duplicate live Copilots unless explicitly requested.
- Do not bind lists, workflows, Agents, HTTP connections, or knowledge sources unless target app resources are explicit.
- Treat exported packages as templates only after validating names, wrapper type, package shape, and instruction quality.
